# MaxBridge

Самохостинговый коннектор: один телефонный номер с тремя мессенджерами
(**MAX**, **Telegram**, **WhatsApp**) → Открытая линия **Битрикс24** → отдел продаж.

Клиент пишет **на номер** в любом мессенджере — сообщение падает в Битрикс,
менеджер отвечает там же, ответ уходит клиенту обратно. Телефон лежит «в шкафу».

Аналог Wazzup/Ebox — только свой, без абонентки. Только VPS + SIM (+GREEN-API для MAX).

---

## Как это работает

```
[WhatsApp · MAX · Telegram]
         │  (userbot-сессии на одном номере)
         ▼
    [core — FastAPI]  ←→  [Битрикс24 Открытая линия]
         │                      ↕ imconnector API
    [SQLite: чаты]         [Операторы / продавцы]
         │
    [Caddy HTTPS]  ←  веб-морда: статус, QR, вход, подсказки
```

Маршрутизацию («только продавцы», «на ответственного если клиент в CRM»)
делает **сама Открытая линия** — код этим не занимается.

## Быстрый старт

> Требуется VPS с белым IP, DNS A-запись на него и аккаунт Битрикс24.

```bash
curl -fsSL https://raw.githubusercontent.com/aasamets/maxbridge/main/install.sh \
  -o /tmp/install.sh && bash /tmp/install.sh
```

Установщик интерактивно спросит домен, данные Битрикс24, VLESS-ссылку и ключи
GREEN-API (для MAX). Соберёт Docker-образы, запустит сервисы, покажет пароль веб-морды.

После установки открой веб-морду и подключи каналы по одному:

### WhatsApp
1. В карточке **WhatsApp** появится QR-код
2. В приложении WhatsApp: три точки → Связанные устройства → Привязать устройство
3. Наведи камеру — статус сменится на «Подключён»

> QR обновляется автоматически каждые 30 секунд. Если не успел — нажми «Обновить QR».

### MAX (через GREEN-API)
1. Зарегистрируйся на [green-api.com/max](https://green-api.com/max) — создай MAX-инстанс
2. Скопируй **ID инстанса** и **Токен** → вставь в Настройки в веб-морде → Сохранить
3. В карточке MAX нажми «Показать QR» → отсканируй в приложении MAX на телефоне

> Бесплатный период, затем ~690 ₽/мес. Управление инстансом — в кабинете GREEN-API.

### Telegram
1. Получи `api_id` и `api_hash` на [my.telegram.org](https://my.telegram.org)
   *(с российского IP напрямую, без прокси; зайди под тем же номером)*
2. Добавь их в Настройки → Сохранить → перезапусти контейнер `telegram`
3. В карточке Telegram введи номер телефона → получи код → введи код
4. Если включён 2FA — введи пароль

### Битрикс24
1. В Битрикс24: Приложения → Разработчикам → Локальное приложение
2. Права: `imconnector, imopenlines, crm, im`. URL обработчика: `https://ВАШ-ДОМЕН/bitrix/events`
3. Скопируй Client ID и Client Secret → Настройки в веб-морде → Сохранить
4. Нажми **Авторизовать приложение** → разреши в Битриксе
5. Привяжи коннектор к Открытой линии в Битрикс24 → Контакт-центр

## Особенности развёртывания в РФ

- **WhatsApp** в РФ не заблокирован — работает напрямую с VPS.
  Если провайдер VPS блокирует WA на сетевом уровне, фолбэк — через тот же VLESS-туннель:
  добавь `WA_PROXY_HOST=xray` в `.env` и пересоздай контейнер `wa`.
- **Telegram** заблокирован — трафик адаптера идёт через Xray (VLESS-Reality → SOCKS5).
  Нужна **VLESS-ссылка** (`vless://...`) от зарубежного сервера с Reality.
  Устанавливать Xray отдельно не нужно — установщик настроит сам.
- **MAX** — российский мессенджер, работает через GREEN-API без прокси.

> `my.telegram.org` открывай с российского IP напрямую (без прокси/VPN) — иначе не откроется.

## Стек

- **Ядро** — Python 3.12, FastAPI, SQLite
- **WhatsApp** — Node.js 20, [@whiskeysockets/baileys](https://github.com/WhiskeySockets/Baileys)
- **MAX** — Python, [max-api-client-python](https://pypi.org/project/max-api-client-python/) (GREEN-API)
- **Telegram** — Python, [Telethon](https://github.com/LonamiWebs/Telethon)
- **Прокси** — [Xray-core](https://github.com/XTLS/Xray-core) (VLESS-Reality, SOCKS5)
- **HTTPS** — [Caddy](https://caddyserver.com) (авто Let's Encrypt)
- **Контейнеры** — Docker Compose (6 сервисов: core, wa, max, telegram, caddy, xray)

## Диагностика

### WhatsApp показывает QR, но после скана «Не подключён»
Повторно отсканируй QR — возможно, истёк пока сканировал. QR живёт ~20 секунд,
адаптер генерирует новый автоматически. Кнопка «Обновить QR» — принудительное обновление.

### WhatsApp не показывает QR (статус «Не подключён», нет картинки)
Провайдер VPS блокирует WhatsApp. Добавь в `.env` на сервере:
```bash
WA_PROXY_HOST=xray
```
Затем пересоздай контейнер (не restart): `docker compose up -d wa`

### Telegram не подключается
- Убедись что `TG_API_ID` и `TG_API_HASH` заданы в Настройках
- Проверь что xray запущен: `docker compose ps xray`
- Проверь прокси: `docker exec maxbridge-core-1 curl -x socks5h://xray:1080 https://github.com`

### MAX — QR недоступен
Проверь ID инстанса и токен в Настройках. Инстанс должен быть создан и активен
в кабинете [green-api.com/max](https://green-api.com/max).

## Предупреждения

- **Серая зона.** Вход под обычным аккаунтом (не официальным ботом) нарушает правила
  WhatsApp, Telegram и MAX. Аккаунт/номер могут заблокировать. Используй в разумных
  пределах: человеческий темп, без рассылок, один номер — один аккаунт.
- **Сессии.** Периодически потребуется переавторизация через веб-морду.
  Держи SIM активной и телефон онлайн.
- **GREEN-API.** Для MAX используется сторонний managed-шлюз. Сообщения проходят
  через их серверы. Бесплатный период ~3 чата, платно ~690 ₽/мес.
