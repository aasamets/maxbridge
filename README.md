# MaxBridge

Самохостинговый коннектор: один телефонный номер с тремя мессенджерами
(**MAX**, **Telegram**, **WhatsApp**) → Открытая линия **Битрикс24** → отдел продаж.

Клиент пишет **на номер** в любом мессенджере — сообщение падает в Битрикс,
менеджер отвечает там же, ответ уходит клиенту обратно. Телефон лежит «в шкафу».

Аналог Wazzup/Ebox — только свой, без абонентки. Только VPS + SIM.

---

## Как это работает

```
[WhatsApp · MAX · Telegram]
         │  (userbot-сессии на одном номере)
         ▼
    [core — FastAPI]  ←→  [Битрикс24 Открытая линия]
         │                      ↕ imconnector API
    [SQLite: чаты]         [Операторы / продавцы]
         │
    [Caddy HTTPS]  ←  веб-морда: статус, QR, вход
```

Маршрутизацию («только продавцы», «на ответственного если клиент в CRM»)
делает **сама Открытая линия** — код этим не занимается.

## Быстрый старт

> Требуется VPS с белым IP, DNS A-запись на него и аккаунт Битрикс24.

```bash
curl -fsSL https://raw.githubusercontent.com/aasamets/maxbridge/main/install.sh \
  -o /tmp/install.sh && bash /tmp/install.sh
```

Установщик интерактивно спросит домен, данные Битрикс24 и ключи Telegram,
соберёт Docker-образы, запустит сервисы и покажет логин/пароль веб-морды.

После установки:

1. Открой веб-морду → **Авторизовать приложение** (OAuth Битрикс24, один раз)
2. В карточке **WhatsApp** отсканируй QR телефоном
3. В карточке **Telegram** введи номер и код из приложения *(нужны `api_id` / `api_hash` с [my.telegram.org](https://my.telegram.org))*
4. В Битрикс24 → Контакт-центр → привяжи коннекторы к своей Открытой линии

## Требования

| Параметр | Минимум |
|---|---|
| ОС | Ubuntu 22.04+ / Debian 12+ |
| CPU | 1 vCPU |
| RAM | 1 ГБ + **2 ГБ swap** (обязательно) |
| Диск | 10 ГБ |
| Docker | 24+ с Compose plugin |
| Домен | A-запись → IP сервера (для HTTPS) |

**Протестировано на:** Ubuntu 26.04 LTS, 1 vCPU, 1 ГБ RAM + 2 ГБ swap, 10 ГБ SSD —
сервис Digital Ocean / аналог; провайдер [FirstVDS](https://firstvds.ru).

## Особенности развёртывания в РФ

- **WhatsApp** и **Telegram** недоступны напрямую с большинства российских VPS.
  Установщик настроит VLESS-туннель через Xray — трафик обоих мессенджеров
  пойдёт через заграничный сервер.
- Нужна **VLESS-ссылка** (`vless://...`) от любого зарубежного сервера с Reality.
  Устанавливать Xray на сервере не нужно — только ссылка.
- **MAX** работает напрямую (российский мессенджер, не заблокирован).

## Стек

- **Ядро** — Python 3.12, FastAPI, SQLite
- **WhatsApp** — Node.js 20, [@whiskeysockets/baileys](https://github.com/WhiskeySockets/Baileys)
- **Telegram** — Python, [Telethon](https://github.com/LonamiWebs/Telethon)
- **Прокси** — [Xray-core](https://github.com/XTLS/Xray-core) (VLESS-Reality, SOCKS5)
- **HTTPS** — [Caddy](https://caddyserver.com) (авто Let's Encrypt)
- **Контейнеры** — Docker Compose

---

## Предупреждения

- **Серая зона.** Вход под обычным аккаунтом (не ботом) нарушает правила WhatsApp,
  Telegram и MAX. Аккаунт/номер могут заблокировать. Минимизируй риск: человеческий
  темп, без рассылок, один номер — один аккаунт.
- **Сессии отваливаются.** Периодически потребуется переавторизация через веб-морду.
  Держи SIM активной и телефон онлайн.

## Структура

```
core/               FastAPI-ядро: /incoming, /bitrix/events, веб-морда
adapters/           Python-адаптеры: Telegram, MAX
wa_adapter/         Node.js-адаптер: WhatsApp (Baileys)
xray/               Конфиг Xray (VLESS → SOCKS5 для мессенджеров)
install.sh          Интерактивный установщик
Caddyfile.template  Шаблон конфига Caddy
docker-compose.yml  Оркестрация сервисов
```
